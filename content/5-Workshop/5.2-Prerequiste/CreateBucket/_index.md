#### Create S3 Bucket

In this step, you will create 3 S3 buckets for the system:

- **Frontend Bucket**: stores the build files for the React frontend.
- **Audio Bucket**: stores the audio/video files uploaded by users.
- **Output Bucket**: stores the transcript results generated by Amazon Transcribe.

### 1. Access Amazon S3

1. Sign in to the **AWS Management Console**.
2. In the search bar, enter **S3**.
3. Select the **S3** service.

![Open S3](/images/5-Workshop/5.2-Prerequisite/CreateBucket/s3-1.png)

### 2. Create the Frontend Bucket

1. Choose **Create bucket**.
2. In **Bucket name**, enter:

```text
frontend
```

3. In **AWS Region**, select the region you are using for the workshop, for example `ap-southeast-1`.
4. In **Object Ownership**, keep the default value.
5. In **Block Public Access settings**, keep all public access blocking options enabled.
6. Keep the remaining settings as default.
7. Choose **Create bucket**.

![Create frontend bucket](/images/5-Workshop/5.2-Prerequisite/CreateBucket/s3-2.png)

### 3. Create the Audio Bucket

1. Choose **Create bucket**.
2. Enter the bucket name:

```text
audio-bucket
```

3. Select the same region as the Frontend Bucket.
4. Keep **Block all public access** enabled.
5. Choose **Create bucket**.

![Create input bucket](/images/5-Workshop/5.2-Prerequisite/CreateBucket/s3-3.png)

### 4. Create the Output Bucket

1. Choose **Create bucket**.
2. Enter the bucket name:

```text
output-bucket
```

3. Select the same region as the other buckets.
4. Keep **Block all public access** enabled.
5. Choose **Create bucket**.

![Create output bucket](/images/5-Workshop/5.2-Prerequisite/CreateBucket/s3-4.png)

### 5. Configure CORS for the Audio Bucket

The frontend will upload files directly to the Audio Bucket by using a presigned URL, so you need to configure CORS for this bucket.

1. Open the `audio-bucket` bucket.
2. Select the **Permissions** tab.
3. Scroll down to **Cross-origin resource sharing (CORS)**.
4. Choose **Edit**.
5. Paste the following configuration:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["PUT", "POST", "GET"],
    "AllowedOrigins": ["*"],
    "ExposeHeaders": ["ETag"]
  }
]
```

6. Choose **Save changes**.

![S3 CORS](/images/5-Workshop/5.2-Prerequisite/CreateBucket/s3-5.png)

### 6. Configure Event Notification for the Audio Bucket

After a user uploads a file, the Audio Bucket will send an event to an SQS Queue so that Lambda can process it asynchronously.

> You can return to this step after creating the SQS Queue in section 5.2.3.

1. Open the `audio-bucket` bucket.
2. Select the **Properties** tab.
3. Scroll down to **Event notifications**.
4. Choose **Create event notification**.
5. Enter the event name:

```text
audio-uploaded-event
```

6. In **Event types**, select **All object create events**.

![S3 event notification](/images/5-Workshop/5.2-Prerequisite/CreateBucket/s3-6.png)

7. In **Destination**, select **SQS queue**.
8. Select the `audio-processing-queue` queue.
9. Choose **Save changes**.

![S3 event notification](/images/5-Workshop/5.2-Prerequisite/CreateBucket/s3-7.png)

### 7. Verify the Result

After completing this step, you should have 3 buckets:

- The Frontend Bucket used to deploy the React app.
- The Audio Bucket used to receive uploaded files.
- The Output Bucket used to store transcripts.

![S3 buckets result](/images/5-Workshop/5.2-Prerequisite/CreateBucket/s3-8.png)
