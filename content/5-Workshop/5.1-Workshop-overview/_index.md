#### Introduction to the AWS Audio To Text project

The AWS Audio To Text project is a serverless system on AWS that allows users to upload audio or video files such as MP3, MP4, WAV, and M4A to automatically convert speech content into text.

Users access the website through Amazon CloudFront, sign in with Amazon Cognito, and then upload files to Amazon S3 using a presigned URL. After upload, the audio file is processed automatically through S3 Event, SQS, Lambda, and Amazon Transcribe. The transcript result is stored in the S3 Output Bucket, the job status is stored in DynamoDB, and the system sends email notifications through Amazon SNS.

#### Workshop overview
- *Web interface*: React is built and stored in the Amazon S3 Frontend Bucket, then distributed through Amazon CloudFront so users can access the system over HTTPS.
- *User management*: Amazon Cognito handles registration, sign-in, and issues JWT tokens for user authentication when calling APIs.
- *API gateway*: Amazon API Gateway receives requests from the frontend, verifies Cognito tokens, and forwards the requests to the appropriate Lambda functions.
- *File upload*: lambda-upload-url generates a presigned URL so users can upload audio/video files directly to the Amazon S3 Audio Bucket, while also creating job information in DynamoDB.
- *Input file storage*: The Amazon S3 Audio Bucket stores MP3, MP4, WAV, or M4A files uploaded by users.
- *Processing queue*: Amazon SQS receives events from the S3 Audio Bucket after files are uploaded, allowing the system to process asynchronously and avoid overload.
- *Conversion processing*: lambda-processor reads messages from SQS and calls Amazon Transcribe to convert audio content into text.
- *Result storage*: The Amazon S3 Output Bucket stores the transcript result, including transcript.json generated by Transcribe and transcript.txt processed for easier reading by users.
- *Result handling*: lambda-result-handler reads the transcript.json file, creates transcript.txt, updates the job status in DynamoDB, and sends notifications via SNS.
- *Status management*: Amazon DynamoDB stores job information, processing status, user_id, file name, creation time, and transcript path.
- *Result retrieval*: lambda-get-job reads the job status from DynamoDB, retrieves transcript.txt from the S3 Output Bucket, and returns the result to the web interface.
- *Conversion history*: lambda-list-jobs retrieves the list of files uploaded and converted for the currently signed-in user.
- *Notifications*: Amazon SNS sends email notifications when the audio-to-text conversion process is completed.
- *System monitoring*: Amazon CloudWatch records logs from the Lambda functions to support inspection, debugging, and monitoring of system activity.
