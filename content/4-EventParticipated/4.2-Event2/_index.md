# Reflection Report: "FCJ Community Day"

### Purpose of the Event

- Share knowledge about AI and Cloud
- Provide a place to connect, exchange ideas, and share knowledge among FCJ members

### Speakers

- **Vinh Tran** - System Administrator at Central Retail Group
- **Viet Phat** - AI majoring at Swinburne University of Technology
- **Truong Phuoc** - Speaker
- **Nguyen Quoc Bao** - Speaker
- **Le Hoang Gia Dai** - Speaker
- **Bao Huynh** - Junior Cloud Native Developer at Endava Vietnam / Founder / Head of Lab at ITea Lab

### Key Highlights

#### A real career journey from IT Helpdesk to Senior Sysadmin and early steps toward Cloud and DevOps

- From IT Helpdesk to Sysadmin: Starting point, core skills, and career milestones
- Life as a System Administrator: Daily responsibilities, operational thinking, and lessons learned from incidents
- My first cloud journey and transition to DevOps: Cloud is not just a new technology, but also a new way of thinking
- My interview journey at Central Retail Group: Application process, interview rounds, and lessons learned
- Lessons and advice: Common mistakes, development strategies, and practical recommendations

#### GRAPHRAG: Building a GraphRAG application with Amazon Bedrock and Amazon Neptune

- Introduction to RAG
- Exploring GraphRAG
- Building a GraphRAG solution with Amazon Bedrock and Neptune
- Conclusion

#### The art of effective teamwork

- Rule 1: Clear and aligned goals
- Rule 2: The right person in the right place
- Rule 3: Open communication and active listening
- Rule 4: Personal responsibility

#### Multiplayer in the Cloud: Connecting Godot client with AWS WebSockets

- Introduction to networking in multiplayer games
- Choosing the architecture for multiplayer games
- Overview of AWS architecture
- Integrating the Godot client
- Demo
- Challenges, lessons learned, and future directions

#### WAF + ML to detect network attacks: A Machine Learning-based Network Intrusion Detection System (NIDS) on AWS

- AWS WAF: Capabilities, strengths, and limitations
- Why signature-based protection is not enough against modern attacks
- Introduction to NIDS and behavior-based detection using Machine Learning
- Building and training an ML-based NIDS using the CSE-CIC-IDS2018 dataset
- Developing a real-time dashboard to monitor and visualize attacks
- Combining NIDS predictions with AWS WAF events to improve threat detection
- System architecture, deployment on AWS, and operating flow
- Demo, lessons learned, challenges, and future improvements

#### Docker – A containerization technology

- Virtualization
- Containerization
- Comparison: Virtualization vs. Containerization
- Introduction to Docker
- Illustration / Demo

### What I Learned

- Through the event, I learned a lot of practical knowledge about career paths in IT, from IT Helpdesk and Sysadmin to Cloud and DevOps. In addition, I gained a better understanding of technologies such as GraphRAG, AWS WebSockets, WAF combined with Machine Learning, NIDS, and Docker in building and operating modern systems.

### Application to Work

- This knowledge can be applied to career planning, improving system operations skills, teamwork, and deploying cloud solutions. In particular, the content on Docker, network security, GraphRAG, and WebSockets provides a solid foundation for developing real-world projects related to AWS and DevOps.

### Experience in the Event

- The event gave me many practical perspectives through shared experiences, system architecture discussions, and technology demos. I found the content very useful, close to my learning direction, and motivating for continuing to grow in Cloud, DevOps, and AI.
