# Reflection Report: "Event 11/07/2026"

### Purpose of the Event

- To share knowledge about code review tools, effective preparation methods before taking AWS Cloud certification exams, and SLA.

### List of Speakers

- **Thinh Nguyen** - DevOps First Cloud AI Journey
- **Nguyen Huynh Son** - Speaker
- **Ngo Le Tuan Huy** - Speaker

### Highlights

#### Securing Your Web Application with AWS Security Agent
- Security Bottlenecks
- Introduction to Frontier Agent
- Security assessment for design and code
- Automated Penetration Testing
- Actual costs
- Limitations of this Agent

#### SLA and Monitoring: From SLA to Monitoring What Truly Matters
- What is an SLA?
- Monitoring as Part of Risk Management
- The Gap Between "Healthy Infrastructure" and "Satisfied Users"
- Monitoring Pyramid

#### Learning About the Exam: AWS Cloud Practitioner
- Content Overview
- Exam Structure
- How Should I Prepare for This Exam?
- Reference Materials
- Tips & Tricks

### Lessons Learned
Through the event, I gained a clearer understanding of how AWS Security Agent supports security assessment from the design stage, source code review, and automated penetration testing, while also becoming aware of the costs and limitations of using this tool. I also learned about the role of SLA, risk management, and how to build a monitoring system focused on real user experience instead of only tracking infrastructure status. In addition, the introduction to the AWS Certified Cloud Practitioner exam helped me understand the exam structure, the topics to review, and effective preparation methods.

### Application to Work
The knowledge about AWS Security Agent can be applied to detect vulnerabilities early in design and source code, thereby reducing the time required for manual security testing in web projects. Knowledge of SLA and the monitoring pyramid helps me choose appropriate metrics to track system performance, service errors, and user satisfaction. At the same time, the AWS Cloud Practitioner exam preparation roadmap helps me strengthen my foundational AWS knowledge and prepare better for Cloud-related work.

### Event Experience
The event provided a lot of practical content, combining application security, system operations, and AWS certification learning guidance. The presentations helped me realize that a stable system requires not only healthy infrastructure but also safety and a good user experience. This was a useful opportunity for me to broaden my knowledge, learn from real-world experience, and define a clearer development path in the Cloud field.
